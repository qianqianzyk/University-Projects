# CS架构人脸追踪系统说明

本模块实现了基于CS（Client-Server）架构的人脸追踪与视频流传输，适用于树莓派小车与PC端协同工作场景。

---

## 使用流程

1. **启动摄像头服务**
   - 在小车（树莓派）上启动 MJPG-Streamer 服务，确保摄像头正常工作。
2. **配置客户端**
   - 修改客户端 `cv2.CaptureVideo()` 语句中的网络流端口号，使其与 MJPG-Streamer 配置一致。
3. **启动程序**
   - 先启动小车端服务器程序，再启动PC端客户端程序，等待通信建立。

---

## 常见问题与解决方案

### 问题1：PC端无画面
- **原因**：摄像头编号或端口号配置错误。
- **排查**：
  - 用PC浏览器访问客户端 `cv2.CaptureVideo` 语句中的网址。
  - 若无法连接，检查端口号和MJPG-Streamer服务。
  - 若能连接但无画面，检查摄像头编号，使用 `ls /dev/vi*` 查看并重新配置 `start.sh`。

### 问题2：客户端程序卡死
- **原因**：摄像头断开或MJPG-Streamer被关闭。
- **排查**：
  - 检查摄像头连接和编号。
  - 确保CPU未被高负载程序占用。

### 问题3：客户端显示主机退出连接
- **原因**：客户端长时间未连接服务器。
- **排查**：
  - 启动服务器后及时运行客户端。

---

## 注意事项

- 本模块需树莓派服务器与PC客户端协同运行，暂不支持与语音模块集成。
- 推荐在局域网环境下使用，确保网络稳定。
